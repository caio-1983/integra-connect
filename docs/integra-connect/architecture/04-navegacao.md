# 04 - Navegação

# Navegação da Plataforma

## Objetivo

A navegação do Integra Connect deve refletir a organização do negócio da
empresa usuária.

Menus representam áreas de trabalho, não tecnologias ou integrações.

------------------------------------------------------------------------

# Princípios

-   Poucos itens de primeiro nível.
-   Linguagem simples e corporativa.
-   Organização por domínio.
-   Crescimento sem ruptura.
-   Consistência em toda a plataforma.

------------------------------------------------------------------------

# Estrutura Principal

``` text
Operações
Atendimento
Clientes
Comercial
Analytics
Administração
```

Cada item representa um módulo independente.

------------------------------------------------------------------------

# Sidebar

A Sidebar é responsável exclusivamente pela navegação global.

Ela nunca deve concentrar ações específicas de um módulo.

## Cabeçalho

A identidade principal deve ser a empresa cliente.

A marca Integra Solutions aparece de forma discreta como provedora da
plataforma.

------------------------------------------------------------------------

# Navegação por Contexto

Dentro de cada módulo poderão existir áreas específicas.

Exemplo:

## Atendimento

-   Conversas
-   Workspace

## Clientes

-   Lista
-   Perfil
-   Histórico

## Comercial

-   Pipeline
-   Oportunidades
-   Propostas

------------------------------------------------------------------------

# Navegação Local

Cada módulo pode utilizar:

-   Tabs
-   Breadcrumbs
-   Menus secundários

Desde que não alterem a navegação global.

------------------------------------------------------------------------

# Workspace

O Workspace utiliza uma navegação contextual composta por abas:

-   Contato
-   Compromissos
-   Comercial
-   Histórico

Essas abas representam perspectivas do mesmo cliente, não módulos
independentes.

------------------------------------------------------------------------

# Terminologia

Preferir:

-   Atendimento
-   Clientes
-   Operações
-   Comercial
-   Analytics

Evitar:

-   Chat
-   Inbox
-   CRM View
-   Lead Manager
-   IA

A terminologia deve ser orientada ao negócio.

------------------------------------------------------------------------

# Escalabilidade

Novos módulos deverão ser adicionados sem alterar a estrutura mental do
usuário.

Exemplos futuros:

-   Automações
-   Base de Conhecimento
-   Integrações
-   Auditoria

------------------------------------------------------------------------

# Regra Permanente

Toda decisão de navegação deve responder:

1.  O usuário sabe onde está?
2.  O usuário sabe para onde ir?
3.  O usuário entende o propósito de cada módulo?

Se qualquer resposta for negativa, a navegação deve ser revista.
