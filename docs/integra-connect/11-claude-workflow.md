# Escopo

Todo prompt definirá:

Arquivos permitidos.

Arquivos proibidos.

Objetivo.

Critérios.

Restrições.

O escopo deve ser respeitado integralmente.

---

# Antes de alterar código

Responder internamente:

Qual é o objetivo?

Quais arquivos devo alterar?

Existe componente reutilizável?

Existe risco de quebrar outro módulo?

---

# Durante a implementação

Modificar apenas os arquivos autorizados.

Evitar alterações colaterais.

Evitar efeitos cascata.

---

# Depois da implementação

Validar:

TypeScript.

Imports.

Componentes.

Build.

---

# Resposta Final

Sempre informar:

Resumo da implementação.

Arquivos alterados.

Componentes reutilizados.

Componentes novos.

Impactos.

Pendências.

Nunca responder apenas:

"Concluído."

---

# Regra de Ouro

Se existir dúvida.

Pergunte.

Nunca assuma comportamento.

Nunca invente requisitos.

Nunca extrapole o escopo.

O objetivo é preservar estabilidade e evoluir continuamente o Integra Connect.
