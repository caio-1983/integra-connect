# PD-001 --- Integra Connect como Produto Proprietário

## Status

Aprovado

## Data

2026-06-27

# Problema

A solução utilizada como ponto de partida possuía identidade,
arquitetura e experiência orientadas a um projeto específico,
dificultando sua evolução como um produto SaaS.

# Contexto

A Integra Solutions decidiu transformar a base existente em um produto
proprietário. Toda implementação futura deverá fortalecer a identidade
do Integra Connect e não a do projeto de origem.

# Decisão

O sistema passa a ser oficialmente denominado **Integra Connect**.

Todas as decisões devem considerar que o produto será comercializado
para múltiplos clientes, segmentos e operações.

# Objetivos

-   Criar uma plataforma SaaS B2B.
-   Possuir identidade própria.
-   Permitir evolução modular.
-   Reduzir customizações específicas.
-   Fortalecer a marca Integra Solutions.

# Consequências

## Positivas

-   Evolução consistente.
-   Maior reutilização.
-   Branding próprio.
-   Melhor escalabilidade.
-   Arquitetura orientada ao produto.

## Negativas

-   Funcionalidades específicas podem precisar ser reavaliadas.
-   Personalizações deixam de orientar o roadmap.

# Princípios Permanentes

-   O produto pertence à Integra Solutions.
-   O cliente compra uma plataforma, não um projeto personalizado.
-   A IA é uma ferramenta, não o posicionamento do produto.
-   Toda funcionalidade deve aumentar clareza, produtividade ou
    percepção de valor.

# Impacto

Esta decisão influencia Branding, UX, Arquitetura, Roadmap, Design
System e futuras Product Decisions.

# Regra Permanente

Sempre que houver conflito entre uma necessidade específica de um
projeto e a evolução do produto, a decisão deverá priorizar a
arquitetura e a visão de longo prazo do Integra Connect.
